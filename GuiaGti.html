Guía de Comandos de Git

Comando Descripción
Básico
git init <carpeta> Crea un repositorio vacío en <carpeta>. Sin argumentos, lo crea en el directorio actual.
git clone <repositorio> Clona un <repositorio> en tu máquina local. Puede ser local o remoto (HTTP o SSH).
git config user.name <autor> Define el <autor> para todas las confirmaciones en el repositorio actual.
git config --global user.name <autor> Define el nombre del <autor> para todas las confirmaciones del usuario actual en el sistema.
git config --global user.email <correo> Define el <correo> del autor para todas las confirmaciones del usuario actual.
git add <carpeta/archivo> Agrega los cambios realizados al área de preparación (staging). Acepta expresiones regulares.
git commit -m "<mensaje>" Confirma los cambios en el área de preparación con un <mensaje> descriptivo.
git status Muestra la lista de archivos con cambios, los que están en staging y los que no tienen seguimiento.
git log Muestra el historial de confirmaciones (formato por defecto).
git diff Muestra los cambios no marcados entre el índice y la carpeta de trabajo.
Deshaciendo Cambios
git revert <confirmacion> Crea una nueva confirmación que deshace todos los cambios realizados en la <confirmacion> específica.
git reset <archivo> Elimina el <archivo> del área de preparación (staging), pero mantiene los cambios en el directorio de trabajo.
git reset --hard Reinicia el área de preparación y el directorio de trabajo para coincidir con la última confirmación, descartando todos los cambios no confirmados.
git reset <confirmacion> Mueve la rama actual hacia la <confirmacion>, reiniciando el área de preparación pero dejando la carpeta de trabajo.
git reset --hard <confirmacion> Igual que git reset --hard pero además mueve la rama a una <confirmacion> anterior.
git clean -n Muestra los archivos que van a ser eliminados de la carpeta de trabajo.
git clean -f Elimina permanentemente los archivos sin seguimiento del directorio de trabajo.
Re-escribiendo el Historial
git commit --amend Reemplaza la última confirmación con los cambios que estén en el área de preparación. Permite editar el mensaje.
git rebase <base> Elimina el <archivo> del área de preparación, dejando el directorio de trabajo sin cambios.
git rebase -i <base> Inicia un rebase interactivo que abre un editor para reorganizar, unir o eliminar confirmaciones.
git reflog Muestra un log de todos los cambios en el HEAD del repositorio local (incluso los borrados).
Ramas
git branch <rama> Crea una nueva <rama>. Sin argumentos, lista todas las ramas locales.
git checkout -b <rama> Crea una nueva <rama> y se posiciona en ella.
git merge <rama> Mezcla la <rama> indicada con la rama actual.
Repositorios Remotos
git remote add <nombre> <enlace> Crea una nueva conexión a un repositorio remoto. <nombre> es el alias (ej. origin).
git fetch <remoto> Obtiene los cambios del repositorio <remoto> sin fusionarlos con las ramas locales.
git pull <remoto> Obtiene una copia del repositorio <remoto> y la fusiona al instante con el repositorio local.
git pull --rebase <remoto> Obtiene una copia del repositorio <remoto> y realiza un rebase en vez de un merge.
git push <remoto> <rama> Envía la <rama> local al repositorio <remoto>.
git push --force <remoto> Fuerza el envío a la rama remota, ignorando cambios incompatibles. ¡Usar con mucho cuidado!
git push --all <remoto> Envía todas las ramas del repositorio local al repositorio <remoto>.
git push --tags <remoto> Envía todas las etiquetas locales al repositorio <remoto>.
Opciones para git log
git log --limit=<limite> Limita el número de logs mostrados. Ejemplo: git log -5.
git log --oneline Condensa cada confirmación en una sola línea.
git log -p Muestra todos los cambios de cada confirmación.
git log --stat Incluye los archivos que se modificaron y el número de líneas que fueron añadidas o borradas.
git log --author="<autor>" Busca las confirmaciones de un <autor> en particular.
git log --grep="<patron>" Busca confirmaciones cuyo mensaje coincida con el <patron> asignado.
git log <desde>..<hasta> Muestra las confirmaciones entre un punto y otro (puede ser un hash, rama, HEAD, etc.).
git log -- <archivo> Muestra solo las confirmaciones del <archivo> especificado.
git log --graph --decorate Muestra un gráfico en texto al lado izquierdo de los mensajes de confirmación.
Opciones para git config
git config --global alias.<alias> <comando> Crea un atajo (<alias>) para un <comando> más largo.
git config --system core.editor <editor> Define el <editor> usado por los comandos para todos los usuarios.
git config --global -e Abre el archivo de configuración global para que pueda ser editado manualmente.
Opciones para git diff
git diff head Muestra las diferencias entre la última confirmación y la carpeta de trabajo.
git diff --cached Muestra las diferencias entre el área de preparación (staging) y la última confirmación.
